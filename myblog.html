1. Discuss the scope of var, let, and const

The keywords var, let and const are used to declare variables. Depending on where it is declared, the scope of a variable declared with var is the function scope or global scope. On the other hand, the scope of variable declared with let or const is the block scope.


With an example the difference between the scope of var and let are shown as follow:-

var a = 10; //a is declared in gloal scope
function result() {
let b = 20; //b is declared in the function scope
console.log(a); // Here we can see that, a is accessible within result()
console.log(b); //Here we can see that, b is accessible within result()
}
result();
console.log(a); // Here we can see that, a is accessible within result()
console.log(b); // Here we will get ERROR!, b isn't accessible within result()


According to the example, we can see that a is accessible with both the global and the function scope, since it was declared with var. On the other hand, since b is declared with let, it is only accessible within the function scope.

Here is an example of a constant variable:

const PI = 3.14; // PI is a constant variable
const PI = 3.14; // PI is a constant variable
function output() {
console.log(PI); // PI is accessible from within output()
// PI = 4.14; // Error!!!!!!!!, PI cannot be reassigned
}
output();

According to the example we can see that, when we declared PI variable with a const, it is accessible within the block scope and we will get error if we reassign the value of the variable PI which is declared with const.

2. The use cases of null and undefined

In javascript, Undefined and Null are used to represent missing and empty data.

Undefined:

  • When a variable has been declared but has not been assigne a value yet, is considered as undifined.
  • When there is no explicit return statement is used, then the default return value of the function becomes undefined.
  • When we declared a variable without assigning a valur to it, then it automatically initialized to undefined.


    • Example:

    let a;
    function favSinger() {}
    console.log(favSinger());

    Here we can see that, a is undefined, and in the output we will get undefined since favSinger() return nothing.


    Null:

    Null can be assigned to a variable as a representation of no value.


  • Null is explicitly used to represent the absence of a value or an empty value.
  • when we want to explicitly represent that a variable, property, or parameter has no meaningful value, we typically use Null.


    • Example:

    let p = null;
    let obj = {
    name: "Ed Sheeran",
    age: null
    };

    Here we can see that, p is explicitly set to null. and the object property age is also intentionally set to null.


    3. What do you mean by REST API?

    The full form of API is Application Programming Interface. REST API is an architectural style for Application Program Interface(API) that uses HTTPS request to access and use data.

    RESTful APIs use standard HTTP methods (GET, POST, PUT, DELETE, etc.) to perform actions on resources. These methods have well-defined meanings:

    • GET: Retrieve data from the server.
    • POST: Create a new resource on the server.
    • PUT: Update a resource on the server (often used to fully replace an existing resource).
    • PATCH: Partially update a resource.
    • DELETE: Remove a resource from the server.
